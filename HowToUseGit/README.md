## 💡 核心概念对比

### Git vs GitHub

| 方面 | Git | GitHub |
|------|-----|--------|
| **类型** | 版本控制系统 | 代码托管平台 |
| **位置** | 本地 | 云端 |
| **主要功能** | 版本控制、分支管理 | 协作、代码审查、项目管理 |
| **独立性** | 可以独立使用 | 需要 Git |
| **费用** | 免费开源 | 有免费和付费版本 |

**简单理解：**
- **Git** 就像一个本地的版本控制工具，帮你管理代码历史
- **GitHub** 就像一个在线的代码存储和协作平台，把代码放在云端，方便团队协作

---

### Clone vs Fork

| 场景 | Clone | Fork |
|------|-------|------|
| **用途** | 团队项目开发 | 参与开源项目 |
| **权限** | 需要推送权限 | 无需原仓库权限 |
| **工作流** | Clone → 分支 → 开发 → PR | Fork → Clone Fork → 分支 → 开发 → PR |
| **远程仓库** | origin（项目仓库） | origin（你的 Fork）+ upstream（原仓库） |
| **适合场景** | 公司项目、团队协作 | 开源贡献、学习他人代码 |

**何时使用：**
- **Clone**：你是团队成员，有直接推送权限 → 查看 [05 - Clone 项目的完整工作流程](./05-Clone项目的完整工作流程.md)
- **Fork**：参与开源项目，没有推送权限 → 查看 [06 - Fork 项目的完整工作流程](./06-Fork项目的完整工作流程.md)

## ❓ 常见问题

### 我应该先学哪个？
建议先学习 Git（01-04），因为 GitHub 是基于 Git 的。掌握 Git 基本概念后，学习 GitHub 会更容易。

### Git 和 GitHub 有什么区别？
- **Git** 是一个软件，用于版本控制
- **GitHub** 是一个网站，提供 Git 仓库托管和协作功能

### 我可以不用 GitHub 吗？
可以！Git 可以独立使用，也可以配合其他平台（如 GitLab、Bitbucket）使用。

### 学习 Git 需要多长时间？
- 基础命令：1-2 天
- 熟练使用：1-2 周
- 精通进阶功能：持续实践

### Clone 和 Fork 有什么区别？
- **Clone**：将仓库下载到本地，适合团队项目
- **Fork**：在 GitHub 上复制仓库到你的账号，适合参与开源项目

详细对比请看：[06 - Fork 项目的完整工作流程](./06-Fork项目的完整工作流程.md)

### PR 合并后，我应该继续在原分支开发吗？
**不推荐！** 应该：
1. 切换回 main 分支
2. 拉取最新代码
3. 删除已合并的分支
4. 创建新分支开发新功能

详细说明请看：
- [05 - Clone 项目的完整工作流程 - PR 合并后怎么办](./05-Clone项目的完整工作流程.md#pr-合并后怎么办)
- [06 - Fork 项目的完整工作流程 - 多次贡献的流程](./06-Fork项目的完整工作流程.md#多次贡献的流程)

### 如何参与开源项目？
1. 学习完 Git 基础教程
2. 阅读 [06 - Fork 项目的完整工作流程](./06-Fork项目的完整工作流程.md)
3. 找一个感兴趣的开源项目
4. 阅读项目的 CONTRIBUTING.md
5. 从简单的 Issue 开始（标签：`good first issue`）
6. 提交你的第一个 Pull Request

## 🎯 日常使用核心命令

**基础操作：**
```bash
git clone <url>              # 克隆项目
git status                   # 查看状态
git add .                    # 添加所有修改
git commit -m "提交信息"      # 提交修改
git push                     # 推送到远程
git pull                     # 拉取远程更新
```

**分支操作：**
```bash
git branch                   # 查看分支
git checkout -b feature/xxx  # 创建并切换分支
git merge branch-name        # 合并分支
git branch -d branch-name    # 删除分支
```

**查看历史：**
```bash
git log                      # 查看提交历史
git log --oneline            # 简洁查看历史
git diff                     # 查看修改
```
